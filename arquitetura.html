Backend (API)

💡 Por que é o maior esforço? Atualmente toda a lógica roda no navegador do usuário e os dados ficam locais. O backend cria o "cérebro" compartilhado da aplicação.

Como funciona

O voluntário aponta a câmera do celular ou tablet para o código de barras do produto. A leitura acontece direto no navegador, sem precisar instalar nenhum aplicativo.

O que é preenchido automaticamente

Ao ler o código, o sistema consulta a API Open Food Facts e preenche automaticamente: nome do produto, marca, categoria e unidade de medida. O voluntário confirma e salva.

Por que é valioso para a ONG

💡 Reduz drasticamente o tempo de cadastro. Em vez de digitar nome, categoria e marca, o voluntário escaneia o código e só preenche quantidade e data de validade.

Roteiro de Implantação

Estimativa de 4 a 6 semanas com 1 desenvolvedor backend experiente em Node.js e PostgreSQL

1
Semana 1 · 3–5 dias
Infraestrutura
  • Criar banco no Supabase e configurar o PostgreSQL
  • Criar repositório no GitHub e conectar ao Vercel
  • Publicar o frontend com domínio ama.org.br
  • Criar conta Railway/Render para o servidor backend
  • Criar conta SendGrid e configurar e-mail de envio
2
Semanas 2–3 · ~2 semanas
Desenvolvimento do Backend
  • Criar servidor Node.js + Express e fazer deploy no Railway
  • Modelar as tabelas do banco (produtos, finanças, usuários)
  • Implementar autenticação com JWT e bcrypt
  • Desenvolver endpoints REST para produtos e financeiro
  • Implementar envio de alertas por e-mail
3
Semana 4 · ~1 semana
Integração Frontend ↔ API
  • Substituir chamadas do localStorage pela API real
  • Adaptar o contexto de login para usar tokens JWT
  • Testar todos os fluxos: cadastro, edição, exclusão, filtros
  • Testar em celular e diferentes navegadores
  • Corrigir bugs de integração
4
Semanas 5–6 · ~1 semana
Testes e Lançamento
  • Migrar dados existentes para o banco de produção
  • Criar usuários iniciais para a equipe da ONG
  • Treinamento rápido com coordenadores e voluntários
  • Monitorar o sistema nas primeiras semanas
  • Coletar feedback e ajustar fluxos
5
Após lançamento · ~1 semana
Código de Barras
  • Instalar biblioteca html5-qrcode no frontend
  • Criar tela de escaneamento com câmera do dispositivo
  • Integrar com a API Open Food Facts para busca de produtos
  • Preencher automaticamente nome, marca e categoria
  • Testar em celular e tablet com produtos reais da ONG
