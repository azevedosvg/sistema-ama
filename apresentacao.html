Backend (API) — A Construir

O "cérebro" compartilhado: servidor, autenticação, permissões e comunicação com o banco de dados

🟢 Node.js + Express

Servidor que recebe pedidos do frontend, executa validações e conecta ao banco de dados.

  • Validações e regras de negócio centralizadas
  • Geração de relatórios em PDF e CSV
  • Agendamento de alertas automáticos
  • Integração com banco e serviços externos
Hospedagem: Railway ou Render · ~R$ 40–80/mês

🔐 Autenticação JWT

Substitui o localStorage atual por login seguro com tokens criptografados.

  • Tokens JWT com expiração em 24 horas
  • Bcrypt para hash seguro das senhas
  • Refresh tokens para sessões longas
  • Proteção de rotas no frontend e backend
Elimina senhas em texto simples no localStorage

👤 Permissões (RBAC)

Cada usuário acessa apenas o que sua função permite.

  • Admin: acesso total ao sistema
  • Coordenador: gerencia estoque e financeiro
  • Voluntário: visualiza e registra entradas
  • Log de auditoria: quem alterou o quê e quando
Permissões verificadas no servidor, não no navegador

🔗 API REST (~30 endpoints)

Interface padronizada para comunicação entre frontend e servidor.

  • CRUD completo: produtos, categorias, finanças
  • Endpoints de usuários e permissões
  • Relatórios, exportação e estatísticas
  • Webhook para integrações futuras
Prazo de desenvolvimento: 2–3 semanas

Banco de Dados, Cache e Notificações

Os serviços que guardam, aceleram e comunicam as informações do sistema

🗄️ Banco de Dados

PostgreSQL
Supabase (gratuito) ou Railway

Onde todos os dados ficam armazenados de forma centralizada e segura. Substitui o localStorage do navegador.

  • Produtos, categorias e estoque
  • Transações e histórico financeiro
  • Usuários, funções e permissões
  • Log de auditoria (quem alterou)
✓ Gratuito até 500 MB · Backup diário automático

⚡ Cache

Redis
Upstash — grátis até 10k req/dia

Memória temporária para dados frequentes. Deixa o sistema mais rápido e gerencia sessões.

  • Sessões de usuário logado
  • Cache de consultas frequentes
  • Melhora performance com muitos usuários
  • Filas de e-mails de alerta
Opcional — pode ser adicionado após a 1ª versão

📧 Notificações

SendGrid
100 e-mails/dia no plano gratuito

Alertas automáticos para coordenadores sem precisar entrar no sistema diariamente.

  • Alertas de produtos vencendo em 7 dias
  • Relatório semanal automático
  • Aviso de estoque crítico ou zerado
  • Boas-vindas a novos usuários
✓ Alta prioridade — recomendado desde o início

Leitor de Código de Barras

Funcionalidade planejada para após o lançamento inicial

📷

Cadastro por câmera

O voluntário aponta a câmera do celular para o código de barras. A leitura acontece direto no navegador, sem instalar nenhum aplicativo.

📦 html5-qrcode — leitura via câmera
🌐 Open Food Facts API — dados do produto
💰 Custo: Gratuito
⏱ Esforço estimado: ~1 semana

📋 Como funciona

O sistema abre a câmera, lê o código EAN e consulta automaticamente nome, marca e categoria em uma base de dados pública.

✍️ Preenchimento automático

  • Nome completo do produto
  • Marca e fabricante
  • Categoria (alimento, limpeza…)
  • Unidade de medida

💡 Por que é valioso para a ONG

Em vez de digitar nome, categoria e marca, o voluntário escaneia o código e só preenche quantidade e data de validade. Ideal para recebimento de grandes doações — reduz drasticamente o tempo de cadastro.

Roteiro de Implantação

Estimativa de 4 a 6 semanas com 1 desenvolvedor backend experiente em Node.js e PostgreSQL

1
Semana 1 · 3–5 dias
Infraestrutura
  • Banco no Supabase + configurar PostgreSQL
  • Repositório GitHub + conectar ao Vercel
  • Publicar frontend com domínio ama.org.br
  • Conta Railway/Render para o backend
  • Conta SendGrid e configurar e-mail
2
Semanas 2–3 · ~2 semanas
Desenvolvimento Backend
  • Criar servidor Node.js + Express
  • Modelar tabelas do banco de dados
  • Autenticação JWT e bcrypt
  • Endpoints REST para produtos e financeiro
  • Envio de alertas por e-mail
3
Semana 4 · ~1 semana
Integração Frontend ↔ API
  • Substituir localStorage pela API real
  • Adaptar contexto de login para JWT
  • Testar todos os fluxos: CRUD, filtros
  • Testar em celular e navegadores
  • Corrigir bugs de integração
4
Semanas 5–6 · ~1 semana
Testes e Lançamento
  • Migrar dados para o banco de produção
  • Criar usuários iniciais para a equipe
  • Treinamento com coordenadores
  • Monitorar nas primeiras semanas
  • Coletar feedback e ajustar
5
Após lançamento · ~1 semana
Código de Barras
  • Instalar biblioteca html5-qrcode
  • Tela de escaneamento via câmera
  • Integrar Open Food Facts API
  • Preenchimento automático de campos
  • Testar com produtos reais da ONG
