Infraestrutura & Camada de Dados

Integrante 9 — onde os dados ficam salvos, os dados de exemplo e o build do projeto

1 Deixe o sistema pronto

  • Rode npm run dev, abra localhost:5173 e faça login com admin@ama.org / admin123.
  • Abra o DevTools (tecla F12) → aba Application → Local Storage. É lá que você mostra a sua parte.
"Eu fiz a fundação do projeto: onde os dados ficam guardados (no navegador), os dados de exemplo que já vêm prontos e a configuração que compila e roda tudo."

2 O que mostrar funcionando demonstre no navegador

Só clicar e narrar o comportamento — sem abrir o código.

1

O "banco de dados" no navegador

No DevTools → Local Storage, mostre as chaves: ama_products, ama_transactions, ama_movements, ama_users, ama_activities…

"Não tem servidor: eu uso o armazenamento do próprio navegador como banco de dados. Cada tipo de dado tem a sua chave."
2

Os dados persistem

Cadastre um produto novo, dê F5 e mostre que ele continua lá. Mostre também a chave ama_products mudando no Local Storage.

3

O sistema nunca abre vazio

Explique que, na primeira vez, o sistema já vem com dados de exemplo (47 produtos e 32 transações) — por isso a demonstração de todo mundo tem conteúdo.

4

(Opcional) O build

No terminal, rode npm run build e mostre que o projeto compila e gera a versão de produção.

3 Prints do código (apoio)

Mostre cada print rapidamente e leia só a legenda — o que aquele trecho faz.

📄 O que faz: é o "banco de dados". Cada tipo de dado tem a sua chave no armazenamento do navegador — as mesmas que aparecem no DevTools.
📄 O que faz: o padrão que se repete para cada entidade — ler, gravar e gerar id. Produtos, transações, movimentações e usuários seguem todos este mesmo molde.
📄 O que faz: a "parte esperta". Quando os dados de exemplo mudam de versão, recarrega só eles — preservando as contas de usuário e o histórico já existentes.
📄 O que faz: o ponto de partida da aplicação e a configuração do build (Vite + React + Tailwind), que compila e serve o projeto.

4 Resumo do que eu fiz

  • Camada de dados (storage.ts) com o CRUD de todas as entidades sobre o localStorage.
  • Dados de exemplo (47 produtos e 32 transações) e migração de versão sem perder usuários/histórico.
  • Ponto de entrada da aplicação e configuração de build (Vite, TypeScript).
  • Padronização de código (ESLint).
